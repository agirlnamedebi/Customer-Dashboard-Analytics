# 🎬 DVDRental Customer Intelligence Dashboard

> Mid Exam Project · Data Science · Streamlit + PostgreSQL + Machine Learning

A multi-page interactive dashboard analyzing the **DVDRental** database across 4 dimensions: customer profiles, geographic distribution, rental activity, and RFM segmentation — built with Streamlit and powered by two ML models.

## 🤖 Machine Learning Models

Two models are used in **Section 03**:

### Model 1 — RandomForest Classifier (`rental_model.pkl`)
- **Purpose:** Classify each customer as High Engagement or Low Engagement
- **Features:** `frequency` (rentals per day), `duration_days`
- **Label:** Customers above median frequency → High Engagement
- **Algorithm:** RandomForestClassifier, 100 estimators, random_state=42

### Model 2 — Linear Regression (`rental_trend_model.pkl`)
- **Purpose:** Forecast daily rental volume for the next 5 months
- **Features:** Day number (numeric index from first rental date)
- **Output:** Predicted rentals/day with 95% confidence band
- **Answers:** "Will rental volume go up or down?"

## ⚙️ Setup & Installation

### Prerequisites
- Python 3.9+
- PostgreSQL with the `dvdrental` database loaded
- Git

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/dvdrental-dashboard.git
cd dvdrental-dashboard
```

### 2. Create a virtual environment
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set up the database
Make sure your PostgreSQL is running with the `dvdrental` database.
Update the connection string in each `.py` file if needed:
```python
# Default connection used in the project:
host     = "localhost"
database = "dvdrental"
user     = "postgres"
password = "YOUR_PASSWORD"
port     = "5432"
```

### 5. Train the ML models
```bash
python train_model.py
```
This generates `rental_model.pkl` and `rental_trend_model.pkl` in the project root.

### 6. Run the dashboard
```bash
streamlit run home.py
```

Open your browser at `http://localhost:8501` 🎉

## 🔑 Key Findings

- **16,048 total rentals** across 599 customers
- Top 10 customers contribute only **~3% of rentals** — flat distribution, no dominant power users
- **29% of transactions on weekends** — proportional to days, not a natural spike → opportunity for promo
- Rental volume peaked in **Aug–Sep 2005** and has been in decline since — LR model confirms downward trend
- **49.2% of customers** predicted High Engagement by RandomForest
- Hidden finding: At-Risk customers have **longer duration (177 days)** than Loyal ones (83 days) — duration alone is a false loyalty signal; frequency is what matters
